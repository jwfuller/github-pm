We started doing this to track work for the FOLIO project, the new OneSearch came along because we were working so closely.

GitHub Structures

  • Users
  • Organizations
  • Repository (repo)
  • Issue
  • Project
  • Everyone signs into GitHub with a Personal Account.
  • Personal accounts are then associated with our Organization Account.
  • A Repository is a location in GitHub where files, code, and Issues are stored and managed.
  • Repositories can be owned by Personal or Organization Accounts.
  • Each Repository has option to enable the Issues feature which enables you to track ideas, tasks, bugs, or feedback.
  • You can use Projects to organize and prioritize Issues.

Project Board and Views

Project boards are a great way to get an overview of what is going on

Swimlanes

Swimlanes represent Status and are customized per Project and do not impact other projects. An Issue could be in the "Waiting on" status in your project and "In Progress" in another.

Views

Filtering

You can filtering Issues by things like Assignee, Label, Milestone, Status. These filters are used Project boards and in the full list of issues for a Repository.
